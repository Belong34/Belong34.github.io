keras.layers.Dense

发表于 2019-11-23 更新于 2019-12-07 分类于 Deeping_learning
本文字数： 2.6k 阅读时长 ≈ 2 分钟

Arguments:

  • units：输出空间的维数
  • activation： 要使用的激活功能。 如果未指定任何内容，则不应用任何激活
  • use_bias：布尔值，是否使用偏置项
  • kernel_initializer：权重矩阵的初始化方法
  • bias_initializer：偏置项的初始化方法
  • kernel_regularizer：权重矩阵的正则化方法
  • bias_regularizer：偏置项的正则化方法
  • activity_regularizer：输出项的正则化
  • kernel_constraint：对权重矩阵的约束项
  • bias_constraint：对偏置项的约束项

    activation

    当我们不用激活函数时，权重和偏差只会进行线性变换。线性方程很简单，但解决复杂问题的能力有限。没有激活函数的神经网络实质上只是一个线性回归模型。激活函数对输入进行非线性变换，使其能够学习和执行更复杂的任务。我们希望我们的神经网络能够处理复杂任务，如语言翻译和图像分类等。线性变换永远无法执行这样的任务。激活函数使反向传播成为可能，因为激活函数的误差梯度可以用来调整权重和偏差。如果没有可微的非线性函数，这就不可能实现。总之，激活函数的作用是能够给神经网络加入一些非线性因素，使得神经网络可以更好地解决较为复杂的问题。

激活函数好或坏，不能凭感觉定论。然而，根据问题的性质，我们可以为神经网络更快更方便地收敛作出更好的选择。

用于分类器时，Sigmoid函数及其组合通常效果更好。

由于梯度消失问题，有时要避免使用sigmoid和tanh函数。

ReLU函数是一个通用的激活函数，目前在大多数情况下使用。

如果神经网络中出现死神经元，那么PReLU函数就是最好的选择。

请记住，ReLU函数只能在隐藏层中使用。

一点经验：你可以从ReLU函数开始，如果ReLU函数没有提供最优结果，再尝试其他激活函数。

softmax

Softmax函数的目标：把向量V各个向量进行指数加权计算，结果每一项都在0-1之间，并且所有数值之和为1

softmax函数最好在分类器的输出层使用。产生one-hot的输出效果。

Relu

relu在x<0时是硬饱和。由于当x>0时一阶导数为1。所以，relu函数在x>0时可以保持梯度不衰减，从而缓解梯度消失问题，还可以更快的去收敛。但是，随着训练的进行，部分输入会落到硬饱和区，导致对应的权重无法更新。我们称之为“神经元死亡”。

ReLU 的输出不是零中心的。

Dead ReLU Problem：ReLU 不适合大梯度传播，因为在前层参数更新以后，ReLU 的神经元有可能再也不能被激活，导致梯度永远都是零。通过合理设置学习速率，这种情况的发生概率会降低。

fx=max(0，u)

elu

是relu激活函数的改进版本，解决部分输入会落到硬饱和区，导致对应的权重无法更新的问题。计算激活函数relu，即max(features, 0)，所有负数都会归一化为0，所以的正值保留为原值不变

优点：在于不受”梯度消失”的影响,且取值范围在[0,+oo)

缺点：在于使用了较大的学习速率时,易受达到饱和的神经元的影响

公式和图像如下：左边缩小方差，右边保持方差；方差整体还是缩小的，而均值得不到保障。

selu

左边缩小方差，右边放大方差，适当选取参数alpha和lambda，使得整体上保持方差与期望。如果选取：

lambda=1.0506，alpha=1.67326，那么可以验证如果输入的x是服从标准正态分布，那么SELU(x)的期望为0，方差为1.

  • 不存在死区
  • 存在饱和区（负无穷时， 趋于 - αλ）
  • 输入大于零时，激活输出对输入进行了放大

linear

我们看到了step函数的问题，梯度为零，在反向传播过程中不可能更新权重和偏差。此时，我们可以用线性函数来代替简单的step函数。函数表达式：f（x）=ax+b

exponential

指数激活函数
tf.keras.activations.exponential(x)

hard_sigmoid

tf.keras.backend.hard_sigmoid(x)

sigmoid的分段线性近似。

速度比 sigmoid 更快。

如果 x < -2.5，返回 0。
如果 x > 2.5，返回 1。
如果 -2.5 <= x <= 2.5，返回 0.2 * x + 0.5。

sigmod

  • 输出非常大或者非常小的时候没有梯度
  • 输出均值非0(导致第一层归一化好，后面归一化不好)
  • exp计算复杂
  • 梯度消失(导数小于1，深的时候梯度会消失)

tanh

  • 没有梯度
  • 输出均值是0
  • 计算复杂

Relu

小心设置learning-rate

  • 不饱和(梯度不会过小)
  • 计算量小
  • 收敛速度快
  • 输出均值非0
  • Dead ReLU 不适合大梯度传播，因为在前层参数更新以后，ReLU 的神经元有可能再也不能被激活，导致梯度永远都是零。通过合理设置学习速率，这种情况的发生概率会降低。

relu变式

  • Leaky-relu->解决第五点MAX(0.01x,x)
  • ELU ->均值更接近0，小于0 计算量大(exp)
  • MAXOUT->relu泛化版本，两个参数,max(w1x+b1,w2x+b2)

sigmoid&&tanh&&Relu

sigmoid被淘汰了

相较于 sigmoid 和 tanh 函数，ReLU 对于随机梯度下降的收敛有巨大的加速作用。

softsign

Softsign 是 Tanh 激活函数的另一个替代选择。就像 Tanh 一样，Softsign 是反对称、去中心、可微分，并返回-1 和 1 之间的值。其更平坦的曲线与更慢的下降导数表明它可以更高效地学习，比tanh更好的解决梯度消失的问题。另一方面，导数的计算比 Tanh 更麻烦；

在实践中，可以深度用softsign替代tanh激活函数
